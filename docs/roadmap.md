# Технический статус и Roadmap

**Технический стек:**
* **Backend:** Nuxt 4 (Nitro engine), CrossWS (WebSockets), Unstorage.
* **Frontend:** Vue 3, Composition API, @vueuse/core (useWebSocket).
* **Integrations:** Dota 2 Game State Integration (GSI), tmi.js (Twitch Messaging Interface).

---

## Этап 1: Интеграция с Game State Integration (GSI) — [DONE/IN REVIEW]
*   [x] Настройка локального конфигурационного файла GSI в директории игры.
*   [x] Реализация HTTP-эндпоинта `POST /api/gsi` для десериализации данных от игрового клиента.
*   [x] Разработка серверной утилиты `server/utils/ws.ts` для широковещательной рассылки данных (broadcast).
*   [x] Инициализация Nitro-плагина для управления WebSocket-соединениями.
*   [ ] **Milestone:** Валидация получения корректного JSON-объекта от Dota 2 и его ретрансляция в сокет.

## Этап 2: Уровень персистентности и Twitch API — [NEXT]
*   [ ] Конфигурация драйвера **Unstorage** (локальное хранилище) для управления состоянием приложения (`appState`).
*   [ ] Интеграция `tmi.js`: создание сервиса для подключения к IRC-серверу Twitch.
*   [ ] Разработка парсера команд чата: `!mode`, `!bet`, `!vote`.
*   [ ] Реализация логики изменения состояния: запись выбранного режима в Unstorage и отправка события `mode_change` через WS.

## Этап 3: Разработка Frontend Overlay
*   [ ] Создание базовой структуры `pages/overlay.vue`.
*   [ ] Реализация реактивного состояния (Reactive State) на основе входящего потока данных из WebSocket.
*   [ ] Разработка компонентов отображения данных (HealthBar, Gold, KDA).
*   [ ] Внедрение логики переключения UI-слоев в зависимости от текущего режима (`tryhard` / `fun`).

## Этап 4: Реализация бизнес-логики интерактивов
*   [ ] **Sweat Meter:** Алгоритм расчета интенсивности игры на основе дельты GPM/XPM за временной интервал.
*   [ ] **Wall of Grief (Стена плача):** Буферизация чата и триггер визуализации при флаге `hero.alive === false`.
*   [ ] **Item Voting:** Логика сбора голосов за предметы и сопоставление с `item_usage` из GSI.

## Этап 5: Визуальные эффекты и оптимизация
*   [ ] Оптимизация рендеринга и минимизация задержек передачи данных.
*   [ ] Внедрение сложных CSS-анимаций и Transition-групп для динамических элементов.
*   [ ] Настройка локального лидерборда на базе Unstorage (Top-3 фанатов матча).

---

## Технические примечания:
*   **Порт приложения:** 3000 (Nuxt + WS).
*   **События сокета:** `update` (статистика), `death` (триггер смерти), `mode_change` (смена режима), `vote_update` (статистика голосов).
